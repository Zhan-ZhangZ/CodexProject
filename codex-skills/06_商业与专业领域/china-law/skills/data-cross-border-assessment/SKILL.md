# /data-cross-border-assessment

评估数据出境合规路径——路径选择、门槛判断、程序指引。

## 指令

### 1. 事务上下文

- 数据处理者身份？（关键信息基础设施运营者(CIIO)/非CIIO）
- 数据出境类型？（个人信息/重要数据/两者皆有）
- 个人信息规模？（处理100万人以上个人信息？累计出境人数？）
- 出境目的地？（是否有充分性认定或等效保护？）
- 出境场景？（业务需要/集团内部管理/法律义务）

### 2. 核心法律框架

#### 2.1 出境路径概述（《个人信息保护法》第38条 `[PKULaw]`）

个人信息处理者因业务等需要确需向中国境外提供个人信息的，应具备下列条件之一：
1. 通过国家网信部门组织的安全评估
2. 经专业机构进行个人信息保护认证
3. 按照标准合同与境外接收方订立合同
4. 法律、行政法规或国家网信部门规定的其他条件

#### 2.2 安全评估——触发门槛（《数据出境安全评估办法》第4条 `[PKULaw]`）

有下列情形之一的，应申报数据出境安全评估：
1. 向境外提供重要数据
2. 关键信息基础设施运营者向境外提供个人信息
3. 处理100万人以上个人信息的数据处理者向境外提供个人信息
4. 自上年1月1日起累计向境外提供10万人以上个人信息或1万人以上敏感个人信息
5. 国家网信部门规定的其他情形

#### 2.3 标准合同——适用条件（《个人信息出境标准合同办法》第4条 `[PKULaw]`）

非CIIO + 处理个人信息未达100万人 + 上年起累计出境未达10万人个人信息/1万人敏感个人信息 → 可通过标准合同备案出境

**标准合同主要内容（第6条 `[PKULaw]`）：**
- 个人信息处理者和境外接收方的基本信息
- 出境个人信息的目的、范围、类型、敏感程度、数量
- 个人信息处理者和境外接收方保护个人信息的责任和义务
- 个人信息主体权益保障
- 救济渠道
- 合同解除和违约责任

#### 2.4 数据出境安全评估程序（《数据出境安全评估办法》）

**申报材料（第6条 `[PKULaw]`）：**
- 申报书
- 数据出境风险自评估报告
- 数据处理者与境外接收方拟订立的法律文件
- 安全评估工作需要的其他材料

**评估流程：**
省级网信办 → 收到材料后5个工作日内完成完备性查验 → 国家网信办 → 45个工作日内完成（复杂可延长）

**评估重点（第8条 `[PKULaw]`）：**
- 出境目的、范围、方式的合法性、正当性、必要性
- 境外接收方的数据保护水平
- 出境数据的数量、范围、种类、敏感程度
- 个人信息权益保障
- 法律文件是否充分约定了数据安全保护责任义务

#### 2.5 促进和规范数据跨境流动规定（2024新规）

**豁免安全评估的情形：**
- 国际贸易、学术合作、跨国生产制造和市场营销等活动中收集的不含个人信息/重要数据的数据
- 境外收集个人信息后在境内处理
- 为订立和履行个人作为一方当事人的合同（如跨境购物、跨境寄递、跨境汇款、跨境支付、跨境开户、机票酒店预订、签证办理等）确需提供个人信息
- 按照依法制定的劳动规章制度和集体合同实施跨境人力资源管理确需提供员工个人信息
- 紧急情况下保护自然人的生命健康和财产安全确需提供个人信息
- 自贸区负面清单外的数据出境

→ 具体适用范围以正式发布文本为准 `[PKULaw]`

#### 2.6 重要数据识别

**《数据安全法》第21条 `[PKULaw]`：** 国家建立数据分类分级保护制度

重要数据由各地区、各部门确定重要数据具体目录 → 处理者应判断是否处理重要数据

#### 2.7 法律责任（《个人信息保护法》第66条 `[PKULaw]`）

违规出境个人信息：
- 责令改正、警告、没收违法所得
- 拒不改正 → 100万元以下罚款 + 直接责任人1-10万元罚款
- 情节严重 → 5000万元以下或上年营业额5%以下罚款 + 停业整顿/吊销许可 + 直接责任人10-100万元罚款

### 3. 出境路径决策

| 条件 | 路径 |
|------|------|
| CIIO + 个人信息出境 | 安全评估（强制） |
| 处理100万人以上PI + 出境 | 安全评估（强制） |
| 上年累计出境10万人PI或1万人敏感PI | 安全评估（强制） |
| 出境重要数据 | 安全评估（强制） |
| 上述都不满足 | 标准合同备案 或 认证 |
| 自贸区负面清单外 | 免予安全评估/标准合同/认证 |

### 4. 输出

出境合规路径判断 + 适用程序指引 + 申报/备案材料清单 + 风险提示 + 决策树。
