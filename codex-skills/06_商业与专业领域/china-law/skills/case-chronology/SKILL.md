# /case-chronology

1. 读取 `~/.claude/plugins/config/claude-for-legal/china-law/CLAUDE.md` → 管辖与法律框架。
2. 从提供的文件中提取所有时间节点和事实。
3. 按时间顺序排列，标注每个条目的证据来源。
4. 标记不确定性、证据状态和缺口。
5. 输出可排序可分类的时间线文件。

---

## Purpose

案件的时间线是诉讼准备的基础——无论是写起诉状、准备证据目录还是准备庭审辩论，你都需要在脑子里有一条清晰的时间线：什么时间、发生了什么事、有什么证据。本技能从案件材料中提取时间节点，构建每条都有证据来源的事实年表。

## 适用场景

- 复杂民事诉讼的时间线梳理
- 合同履行全过程的节点整理
- 劳动争议中从入职到离职的关键事件时序
- 尽调审查中的历史事件梳理
- 知识产权侵权的时间节点确认（在先使用、抢注时间等）
- 多当事人多事件的并行事态整理

## Step 1 — 收集材料

> 请上传或提供需要梳理时间线的相关材料：
> - 合同/协议文本
> - 双方往来邮件和函件
> - 微信/即时通讯记录
> - 付款凭证/银行流水
> - 相关事件的说明或备忘录
> - 起诉状/答辩状等诉讼文书
> - 对方的主张或文件
>
> **需要重点关注的时间节点？** （如：合同签订日期、违约发生时点、解除通知日期、最后履行日期等）

---

## Step 2 — 提取与整理

### 逐项提取

对每一份文件逐一提取：

- 文件中提到的**具体日期**（YYYY-MM-DD）
- 该日期发生的**事件**（客观描述，不评价）
- 涉及的**当事人/相关方**
- **证据来源**（文件名称 + 页码/段落）
- **确定性标记**（"明确日期"/"当事人声称的日期，未核实"/"仅知道月份，日不确定"）

### 不明确日期的处理

| 描述 | 标记 | 时间线中的排列 |
|------|------|-------------|
| 确切日期已知 | 直接列出 | 按日期排列 |
| 只知道月份 | `[月]` | 排在该月所有已知日期事件之后 |
| 只知道年份 | `[年]` | 排在该年末 |
| 知道大致顺序但日期不详 | `[日期待核实 — 事件发生在X之后、Y之前]` | 在X与Y之间 |
| 对方声称但无证据支撑的日期 | `[对方主张 — 未经核实]` | 列入但标注 |
| 日期从文件中间接推断 | `[推断 — 基于XXX]` | 列入但标注 |

## Step 3 — 编排时间线

### 单当事人时间线格式

```
案件大事记 — [案件名称]

编制日期：YYYY-MM-DD
编制依据：[文件清单]

──────────────────────────────────────────────

YYYY-MM-DD | [事件描述]
           | 证据：[文件名称，页码/段落]
           | 类型：[合同签订/履行行为/违约事件/通知/付款/会议/其他]
           | 当事人：[当事人名称]

YYYY-MM-DD | [事件描述]
           | 证据：[文件名称，页码/段落]
           | 类型：[...]
           | 当事人：[当事人名称]

YYYY-MM-DD (月) | [仅知道月份的事件]
                | 证据：[...]
                | 类型：[...]
                | [日期待核实]

──────────────────────────────────────────────
```

### 多当事人并行时间线格式（适用于双方/多方）

```
案件大事记（多当事人视图） — [案件名称]

         甲方（[名称]）           |  日期            |  乙方（[名称]）
─────────────────────────────────┼─────────────────┼─────────────────────────────────
[甲方的行动]                    | YYYY-MM-DD      | [乙方的行动]
证据：[...]                     |                 | 证据：[...]
─────────────────────────────────┼─────────────────┼─────────────────────────────────
                                | YYYY-MM-DD      | [乙方的行动]
                                |                 | 证据：[...]
─────────────────────────────────┼─────────────────┼─────────────────────────────────
[甲方的行动]                    | YYYY-MM-DD      |
证据：[...]                     |                 |
```

## Step 4 — 关键时间节点标注

在时间线中，使用标签标注法律意义上的关键节点：

| 标签 | 含义 | 说明 |
|------|------|------|
| **[合同签订]** | 合同成立和生效的时间点 | 《民法典》第490、502条 `[PKULaw]` |
| **[履行到期]** | 约定的履行期限 | 用于计算迟延履行的时间起点 |
| **[违约发生]** | 一方开始违约的时间 | 违约责任的起算点 |
| **[解除通知]** | 一方发出解除通知的日期 | 《民法典》第565条 `[PKULaw]` |
| **[诉讼时效起算]** | 诉讼时效开始计算的时间 | 《民法典》第188条 `[PKULaw]` |
| **[时效中断]** | 诉讼时效中断的时间 | 《民法典》第195条 `[PKULaw]` — 如发送律师函等 |
| **[起诉日期]** | 向法院递交起诉状的日期 | 诉讼时效是否已过的基准日 |
| **[管辖权异议]** | 管辖权异议提出的时间 | 不迟于答辩期届满前 |
| **[证据交换]** | 举证期限届满日 | 逾期的证据可能不被采纳 |
| **[首次使用]** | 在先使用的起算点 | 知识产权争议（商标在先使用、先用权等） |
| **[抢注日期]** | 商标/域名被他人抢注的日期 | 《商标法》第32条 `[PKULaw]` |
| **[清算开始/结束]** | 公司清算的起止时间 | 《公司法》第229-242条 `[PKULaw]` |

## Step 5 — 缺口与矛盾标注

### 时间线缺口

列出来证据未覆盖的时间段：

```
时间线缺口

| 时间段 | 应该发生什么 | 为什么重要 | 获取途径建议 |
|--------|------------|----------|------------|
| YYYY-MM-DD至YYY-MM-DD | [推测的活动] | [该段时间涉及……] | [可以通过XX文件/证人补充] |
```

### 时间线矛盾

如果双方当事人对同一事件的主张存在时间矛盾：

```
时间线矛盾

| 事件 | 甲方主张时间 | 乙方主张时间 | 证据比对 | 审查意见 |
|------|------------|------------|---------|---------|
| [事件] | YYYY-MM-DD（证据A） | YYYY-MM-DD（证据B） | [比对结果] | [哪方的主张更有依据] |
```

## Step 6 — 分析应用

### 基于时间线的分析

```
时间线分析要点

1. 时间顺序逻辑检查：
   - [是否存在时间倒置？如：解除通知日期早于违约发生日期 → 问题]
   - [是否存在不可能的时间间隔？]
   - [事件之间的因果关系是否在时间上成立？]

2. 关键期限计算：
   事项\计算\结果\依据
   诉讼时效\从[起算点]至[起诉日]\已过/未过，剩余/超出N天\《民法典》第188条
   除斥期间\从[起算点]至[行使日]\已过/未过\[依据]

3. 时间线与法律适用：
   - [合同成立日在民法典生效前还是后？]
   - [侵权行为发生在何时，新旧法的时间适用范围？]
   - [法不溯及既往的例外情形是否适用？]
```

## 注意事项

- 时间线中的每一条目必须有证据来源——无证据只凭当事人回忆的时间节点应当明确标注 `[当事人陈述 — 未经文件核实]`。
- 日期格式统一为 YYYY-MM-DD，中国日期格式（"2026年5月20日"）可在括号中附注。
- 农历日期须转换为公历日期——如涉及中国农历节日或传统日期，注明转换依据。
- 中国时区和日期标准（GMT+8，公历）→ 涉及跨国交易时注意日期差。

**参考法律渊源:**
- 《中华人民共和国民法典》第188、195、490、502、565条 `[PKULaw]`
- 《中华人民共和国商标法》第32条 `[PKULaw]`
- 《中华人民共和国公司法》(2023修订) 第229-242条 `[PKULaw]`
