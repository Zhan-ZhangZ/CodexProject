# /cold-start-interview

运行冷启动面谈。首次运行写入 `~/.claude/plugins/config/claude-for-legal/china-law/CLAUDE.md`；
后续 `--redo` 运行可重新面谈并在覆盖前显示差异。

## 指令

1. **检查当前状态:** 读取 `~/.claude/plugins/config/claude-for-legal/china-law/CLAUDE.md`。
   如果含 `[待填写]` 或模板占位符，进行全新面谈。如果已填写且未传 `--redo`，
   询问："看起来你已经设置好了。想要重新面谈吗？这将覆盖当前档案（我会先展示差异）。"

2. **遵循下方的面谈脚本。**

3. **要求种子文件:** 请求5-10份近期签署的合同或法律意见书（越多越好，20份可给出更清晰的模式）以及（如存在）审批矩阵。

4. **读取种子文件** 并提取实际的实务立场。

5. **迁移:** 如果 `~/.claude/plugins/cache/claude-for-legal/china-law/*/CLAUDE.md` 存在已填写版本但不在配置路径，复制至配置路径并告知用户。

6. **写入 `~/.claude/plugins/config/claude-for-legal/china-law/CLAUDE.md`**（按需创建父目录），依照以下结构。尽可能使用律师的原话。

7. **展示摘要 + 建议下一步:**
   - "这是我听到的内容——`CLAUDE.md` 已写入。哪里有误？"
   - 提供测试审查："想扔一份合同/劳动问题给我试试吗？"
   - 如果北大法宝已连接：提供验证检索测试

## `--check-integrations`

仅重新运行集成可用性检查（北大法宝、合同管理系统、电子签章、文档存储、飞书/Slack）并更新 `## 可用集成` 节。不重新面谈。

## 示例

```
/china-law:cold-start-interview
```

```
/china-law:cold-start-interview --redo
```

```
/china-law:cold-start-interview --check-integrations
```

```
/china-law:cold-start-interview --area 劳动
```

## 目的

你是第一次见到这个中国法律团队。你的工作是了解**他们**如何做中国法律实务——而非中国法律实务的抽象概述——并将你学到的东西写入一个活文档（插件配置），本插件中的其他每项技能在行动前都会读取它。

律师离开这段对话时应该感觉自己刚对接了一位犀利的助理，问的问题全部恰当。他们应该永远不会看到配置文件。他们应该看到的是一份关于他们团队的文件，可以用自然语言编辑。

## 面谈分步脚本

### 第0步：检查共享公司档案

查找 `~/.claude/plugins/config/claude-for-legal/company-profile.md`。
- **如存在:** 读取。展示一行确认。如确认，跳过公司问题。
- **如不存在:** 在定位和分叉后，询问公司问题并写入共享档案，然后继续插件特定问题。

### 第1步：定位 — 执业场景

在询问任何问题之前，先展示分叉前提示：

> **`china-law` 是为在中国大陆从事法律实务的人设计的——企业法务、执业律师、法律诊所、法学院。** 不在你的领域？`/legal-builder-hub:related-skills-surfacer`。
>
> **2分钟** 完成角色、执业场景、管辖地、主要实务领域的设置。**15分钟** 添加审批矩阵、合规立场、风险容忍度及从签署文件中提取的实务立场。

然后开始：

**必问问题:**
1. 你的角色？执业律师 / 有律师支持的非律师 / 无律师支持的非律师
2. 执业场景？企业法务（单一公司）/ 律所（多客户）/ 法律诊所 / 法学院
3. 主要实务领域？（多选：劳动用工 / 商业合同 / 数据合规 / 知识产权 / 公司治理 / 争议解决 / 综合）
4. 如果企业法务：公司名称？实体类型？（有限责任公司/股份有限公司/外商投资企业/代表处）
5. 如果企业法务：行业？
6. 对接律师是谁？（如果是非律师角色）

### 第2步：集成检测

逐一检测每项集成——仅当 MCP 工具调用实际成功时报告 ✓。已配置但未测试的连接标记为 ⚪。

### 第3步：实务领域配置（按领域）

按用户选择的实务领域逐一配置：

**劳动用工:**
- 劳动合同签订率？使用标准模板还是自行拟定？
- 竞业限制政策？（限制范围、补偿标准——司法解释规定解除前12个月平均工资的30%、违约金额度）
- 解除劳动合同的审批流程？
- 社保和公积金缴纳基数？（实际工资还是最低基数）
- 劳动争议处理策略？（仲裁优先/调解优先/诉讼）

**商业合同:**
- 主要合同类型？（采购/销售/服务/许可/租赁/NNN）
- 交易主体地位？（销售方/采购方/双方皆有）
- 责任上限立场？（直接损失上限为合同金额的N倍/实际损失/法定）
- 管辖地和争议解决偏好？（甲方所在地法院/仲裁——具体仲裁机构）
- 违约金条款立场？（民法典第584条：可预见规则——实际损失为限，不超损失30%为参考）

**数据合规:**
- 是否涉及个人信息处理？
- 是否涉及数据跨境传输？（PIPL第38条路径：安全评估/标准合同/认证）
- 是否属于关键信息基础设施运营者？
- 是否有数据保护官或负责人？

**知识产权:**
- 核心知识产权类型？（专利/商标/著作权/商业秘密）
- 是否有职务发明制度？
- 知识产权归属约定？（甲方/乙方/共同/按贡献）

**公司治理:**
- 公司类型？（有限责任公司/股份有限公司/外商投资企业）
- 股东协议常见条款？
- 是否有股权激励计划？

**争议解决:**
- 争议类型偏好？（诉讼/仲裁/调解）
- 常用仲裁机构？（CIETAC/SHIAC/SCIA/北京仲裁委等）
- 是否有外部合作律所？

### 第4步：审批矩阵

- 审批层级及金额阈值
- 自动升级事项（不受金额限制）

### 第5步：文书风格

- 合同语言？（中文/中英双语/英文为主）
- 审查意见呈现方式？（修订模式批注/单独审查报告/邮件摘要）
- 文书模板偏好？

### 第6步：写入档案

写入 `~/.claude/plugins/config/claude-for-legal/china-law/CLAUDE.md`，创建必要的父目录。

### 第7步：展示摘要 + 提议下一步

展示完整摘要，问"哪里有误？"，提供测试。
