# Reference Boards Registry

287 boards across ~267 sources (265 GitHub repos + 2 ZIP downloads) from OSHW organizations, hacker hardware, 3D printer controllers, CM4/CM5 carriers, ham radio SDR, CNC controllers, drone FCs, eurorack synth modules, and more. Three schematic formats: KiCad 6+/8/9 S-expression, KiCad 5 legacy text, and Eagle XML.

## All Parseable Boards (287 boards)

*Sorted by organization, then board name. Format: `kicad6` (KiCad 6), `kicad7` (KiCad 7), `kicad8`, `kicad9`, `kicad_legacy` (KiCad 4/5), `eagle` (Eagle XML). KiCad 6/7/8/9 all use S-expression format (same parser).*

*Format breakdown: 85 kicad7, 27 kicad8, 30 kicad9, 16 kicad6, 45 kicad_legacy, 84 eagle = 287 total.*

*All Eagle boards use embedded `<libraries>` in .sch XML — no external .lbr needed.*

## Notes

- Adafruit ESP32-S3 Feather: correct repo is `adafruit/Adafruit-Feather-ESP32-S3-PCB`
- Prefer KiCad 6+ over legacy when both exist (ESP32-POE, ANAVI Macro Pad, PocketBeagle)
- Neo6502pc is 1 board but 2 KiCad projects (CPU+PWR) -> 2 registry entries
- Absmach S0 and Wio LR1121 use KiCad 9 — same S-expression parser as KiCad 6+, just newer version field
- ANAVI Fixed PD uses KiCad 8 — also same S-expression parser
- Blues note-hardware: all 4 KiCad boards are KiCad 7 ports by H. Raftery from OrCAD/Altium originals. The remaining ~30 boards in the repo are OrCAD .dsn, Altium .SchDoc, or PDF-only.
- Blues Notecarrier-B is deprecated but fully functional KiCad 7 — **tier 1 board** despite deprecation, valuable as simpler MCP73831-based charger reference vs BQ24210 on A/F
- Cynthion has 17 sheets — deepest hierarchy in the set (surpasses Bus Pirate Ultra's 12)
- USB Armory mark-two-lan is the KiCad 6+ version; mark-two (original) is KiCad legacy. Use mark-two-lan.
- HackRF One, Cynthion, Glasgow, USB Armory are the "marquee" hacker boards — complex, well-maintained, professional quality
- VESC 4.12 uses older KiCad format (pre-KiCad 5 `.sch` files, not `.kicad_sch`) — uses kicad_legacy parser path. VESC 6 layout is NOT open-source (schematic-only), so we use VESC 4.12 which has full schematic + PCB + gerbers
- moteus c1 has multiple hardware revisions in `hw/c1/r1.0/`, `hw/c1/r1.1/`, `hw/c1/r1.2/` — use latest
- HadesFCS has multiple board variants: Hades (main, 5 sheets), HadesMicro, HadesMicroJLCPCB, HadesNT, HadesGPSDongle — register main Hades only
- PX4/Hardware contains 3 boards: FMUv2 (Eagle), Sapog ESC (Eagle), PX4FLOW (Eagle). FMUv3+ are Altium (not parseable). Clone repo once, reuse for all 3.
- Cheap FOCer 2 is a VESC 6-derived design — has spaces in schematic filename: `Cheap FOCer 2 60mm.sch`
- stmbl has both Eagle (`hw/eagle/`) and KiCad (`hw/kicad/`) variants — use KiCad v4.0 for consistency
- Twisted Fields has 5 board designs (base, driver, encoder, UI, debugger) — register only base + driver module (the interesting ones)
- Corne v4 uses hierarchical sub-sheets in `pcbs/common/` shared between cherry/chocolate variants — follow references from corne-cherry top-level
- Ferris 0.2 has multiple sub-variants: compact, bling, mini, high — register compact (simplest, still has all key ICs)
- EEZ H24005 has both Eagle and KiCad files in `Consolidated/` — use Eagle (primary format, `EEZ PSU consolidated r5B13a.sch`, 2.5 MB, 12 internal sheets). KiCad conversion exists at `Consolidated/KiCad/`
- Radar Speed Sign split into 2 entries (controller + HB100 amplifier) — separate KiCad projects in separate directories, independent PCBs
- OpenInverter split into 2 entries (mainboard + gate driver) — sensor_board3 and base_board6 omitted (simpler utility boards). Clone repo once for both.
- SimpleFMCWRadar split into 2 entries (RF board + GainBlock) — separate Eagle boards with separate .brd files
- Injection Molder split into 3 entries (triac + zero-cross + temp sensor) — 5 utility boards omitted (buffer, shift register, decoder, supply, valve controller). 8 total KiCad projects in 8 separate directories.
- moco FOC split into 3 entries (rd501 + od501 + oi401) — most distinct variants. 4-5 other variants omitted (bkd8316, pg4, rc1, rd321, sc1).
- PocketPD hardware is in `CentyLab/PocketPD_HW` (separate from the firmware repo `CentyLab/PocketPD`)
- ExpressLRS-Hardware repo has mixed formats (Eagle, EasyEDA, gerber-only) — only the RX_20x20 v2.0 has parseable Eagle .sch
- emonTx v4 has spaces in filename: `emonTx V4.0.5.sch`
- Crazyflie has a typo in filename: `Crazyflie contol board.sch` (missing 'r' in control) — use as-is
- ODRI micro_driver is deeply nested: `electronics/micro_driver_electronics/eagle_design_files/microDriver_v2.sch`
- tinyPEPPER and tinyFISH are from the same author (fishpepper) but different repos
- HackRF Pro is in a **separate repo** (`hackrf-pro`) from HackRF One (`hackrf`). Codename "praline". 12-sheet hierarchy with dedicated FPGA, mixer, clock, and image reject filter sheets. KiCad 9 format.
- Bus Pirate 6 REV2B is in the `BusPirate5-hardware` repo (name says "5" but contains 6/7 revisions too). KiCad 9 format (version 20250114).
- iCEBreaker-Bitsy shares the `icebreaker-fpga/icebreaker` repo with the full iCEBreaker — both are separate board designs
- GreatFET Azalea (KiCad 4 legacy) and Foxglove (KiCad 5 legacy) are separate main board revisions in `greatfet-hardware`. Azalea is flat (1 sheet), Foxglove has hierarchy (3 sheets).
- Black Magic Probe is **gEDA format**, not KiCad — removed from parseable boards
- ChameleonMini RevG (Eagle 6.6) is a 743 KB single-sheet schematic — large but parseable
- RPi CM4 IO Board and CM5 IO Board are **ZIP downloads**, not GitHub repos. Pipeline needs `wget` + `unzip` instead of sparse git checkout. CM5: `https://pip-assets.raspberrypi.com/categories/1098-design-files/documents/RP-008099-DD-1-CM5%20IO%20Board,%20revision%202,%20KiCAD%20files..zip` → `CM5IO.kicad_sch`. - CM5 Minima REV2 has 8 hierarchical sheets including a LoRaWAN sheet — most complex CM4/CM5 carrier in the set
