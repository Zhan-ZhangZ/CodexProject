# /pipl-review

审查个人信息处理活动的合规性，覆盖告知同意、跨境传输、敏感信息、DPA、PIA。

## 指令

### 1. 事务上下文
- 个人信息处理角色？（个人信息处理者/受托处理者/共同处理者）
- 处理场景？（用户注册/员工管理/客户关系/数据分析/算法推荐/人脸识别）
- 是否涉及敏感个人信息？
- 是否涉及跨境传输？
- 是否涉及自动化决策或算法推荐？

### 2. 核心法律框架

#### 2.1 域外适用（《个人信息保护法》第3条 `[PKULaw]`）

在中华人民共和国境内处理自然人个人信息的活动 → 适用PIPL。

在境外处理境内自然人个人信息的活动，有下列情形之一的 → 也适用PIPL：
- 以向境内自然人提供产品或服务为目的
- 分析、评估境内自然人的行为
- 法律、行政法规规定的其他情形

#### 2.2 告知同意机制（PIPL 第13-18条 `[PKULaw]`）

**合法性基础（第13条）：**
- 取得个人同意
- 为订立或履行个人作为一方当事人的合同所必需
- 履行法定职责或法定义务所必需
- 应对突发公共卫生事件或保护生命健康和财产安全所必需
- 在合理范围内处理已公开的个人信息
- 法律、行政法规规定的其他情形

→ **同意是最后的选择，但实践中是最常用的基础。**

**告知要求（第17条）：**
- 个人信息处理者的名称或姓名和联系方式
- 处理目的、处理方式、处理的个人信息种类、保存期限
- 个人行使权利的方式和程序

**单独同意情形（第29条）：**
- 处理敏感个人信息
- 向第三方提供个人信息
- 公开个人信息
- 向境外提供个人信息
- 公共场所采集的个人图像/身份特征信息用于维护公共安全以外的目的

**禁止强制同意（第16条）：**
不得以个人不同意处理其个人信息或撤回同意为由，拒绝提供产品或服务；
处理个人信息属于提供产品或服务所必需的除外。

#### 2.3 敏感个人信息（PIPL 第28-32条）

定义：一旦泄露或非法使用，容易导致自然人的人格尊严受到侵害或人身、财产安全受到危害的个人信息。包括：
- 生物识别信息（人脸、指纹）
- 宗教信仰、特定身份
- 医疗健康、金融账户
- 行踪轨迹
- 不满14周岁未成年人的个人信息

处理要求：
- 具有特定的目的和充分的必要性
- 采取严格保护措施
- 取得个人的单独同意
- 进行个人信息保护影响评估

#### 2.4 跨境传输（PIPL 第38-43条 `[PKULaw]`）

**路径（第38条）：**
1. 通过国家网信部门组织的安全评估（CIIO + 处理100万人以上个人信息 + 累计10万人个人信息或1万人敏感个人信息出境）→ 数据出境安全评估办法第4条 `[PKULaw]`
2. 经专业机构进行个人信息保护认证
3. 按照国家网信部门制定的标准合同与境外接收方订立合同
4. 其他规定条件

**告知要求（第39条）：**
境外接收方的名称或姓名、联系方式、处理目的、处理方式、个人信息种类、个人行使权利的方式和程序 → 个人单独同意

**数据本地化（第40条）：**
CIIO 和处理个人信息达到国家网信部门规定数量的处理者 → 境内收集和产生的个人信息存储在境内 → 确需出境需安全评估

#### 2.5 数据处理协议（DPA）审查要点

- 是否明确处理者/受托处理者角色？
- 受托处理者的处理目的和方式是否限定？
- 是否禁止转委托？（PIPL第21条：未经同意不得转委托）
- 安全措施是否充分？
- 数据处理后是否要求返还或删除？
- 是否配合个人信息保护影响评估和审计？
- 是否约定协助处理个人行使权利？

#### 2.6 个人信息保护影响评估（PIA）（PIPL 第55-56条）

必须进行PIA的情形：
- 处理敏感个人信息
- 利用个人信息进行自动化决策
- 委托处理个人信息、向第三方提供或公开个人信息
- 向境外提供个人信息
- 其他对个人权益有重大影响的处理活动

PIA内容：
- 处理目的、处理方式等是否合法、正当、必要
- 对个人权益的影响及安全风险
- 保护措施是否合法、有效并与风险程度相适应
