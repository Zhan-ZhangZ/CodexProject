# Contributing

Thanks for improving this skill.

## What Helps Most

- stronger examples for planning, drafting, or review
- sharper guidance about what counts as a concrete narrative finding
- better boundaries between this skill and project-governance skills
- public-safe reference material that does not depend on private manuscripts
- packaging and installation improvements that reduce friction

## Contribution Rules

- Keep the installable package inside `novel-writing/`.
- Keep root files human-facing and package files skill-facing.
- Do not add private manuscript excerpts unless you fully own them and want them public.
- Prefer concrete prompts and diagnosis patterns over generic writing advice.
- If you change workflow rules, update examples and root documentation in the same change.

## Pull Request Checklist

- explain the narrative problem or ambiguity being fixed
- keep the scope narrow
- update docs if behavior or packaging changed
- verify the public package still installs cleanly from the `novel-writing/` path

## Issues

Open an issue when reporting:

- install or upgrade problems
- trigger misfires
- weak or vague review behavior
- unclear chapter-planning guidance
- ideas for high-value additions
