# Xiaohongshu Hot Copywriter

## Overview

这个 skill 用于运行本地小红书 MCP 二进制程序，在首次运行时处理账号登录，抓取当日热门内容，提炼趋势信号，然后主动追问两个关键信息后再生成产品营销推文。

二进制查找顺序：
- 优先使用 skill 目录下的 `bin/`
- 其次使用环境变量 `XHS_MCP_DIR`、`XHS_LOGIN_BIN`、`XHS_SERVER_BIN`
- 再其次尝试系统 `PATH`
- 都找不到则触发自动下载/构建（见下）

如果上述位置都找不到程序：
- 脚本默认会自动尝试从 GitHub Releases 下载当前平台的预编译包（首次使用需联网）
- 如果下载失败，会回退到源码拉取并本地构建（需要 git + go）
- 默认仓库：`https://github.com/xpzouying/xiaohongshu-mcp.git`

附带辅助脚本：
- `scripts/start_xhs_mcp.sh`

## 工作流程

### 1. 检查本地前置条件

在执行任何操作前：
- 优先确认 skill 自带 `bin/` 是否存在可执行文件；如果没有，再使用 `XHS_MCP_DIR` 或系统 `PATH`。
- 优先使用辅助脚本来启动：

```bash
./scripts/start_xhs_mcp.sh status
./scripts/start_xhs_mcp.sh install
./scripts/start_xhs_mcp.sh login
./scripts/start_xhs_mcp.sh server
```

- 脚本默认使用端口 `18060`，并会自动探测常见的 macOS 浏览器安装位置。
- 如果未找到二进制，`login` 和 `server` 也会触发自动安装流程。

### 2. 处理首次登录

如果这是首次运行，或者 MCP 调用失败的原因是小红书会话尚未登录：
- 运行 `./scripts/start_xhs_mcp.sh login`
- 提示用户在打开的浏览器窗口中完成小红书账号登录
- 登录完成后，再运行 `./scripts/start_xhs_mcp.sh server`

不要假设登录状态已经存在。如果环境尚未认证，必须停止后续流程，并先要求用户完成登录。

### 3. 启动本地 MCP 服务

使用以下命令启动主服务：

```bash
./scripts/start_xhs_mcp.sh server
```

预期默认行为：
- HTTP server listens on `http://127.0.0.1:18060`
- MCP endpoint is `/mcp`

如果服务已经在运行，则直接复用。不要在同一端口启动多个实例。

### 3.1 自动安装与回退

当脚本发现本地不存在可执行文件时，按如下顺序处理：
- 先按当前系统平台下载 GitHub Release 产物
- 平台优先覆盖：`darwin-arm64`、`darwin-amd64`、`linux-amd64`、`windows-amd64`
- 若 Release 下载失败，则尝试 `git clone` 仓库并用 `go build` 构建

常用环境变量：
- `XHS_AUTO_INSTALL=true|false`
- `XHS_INSTALL_DIR`
- `XHS_GIT_REPO`
- `XHS_VERSION`
- `XHS_RUNTIME_DIR`

### 4. 使用小红书 MCP 获取热门内容

使用可用的小红书 MCP 工具获取当天的最新热门内容。该程序包含 feed 列表、搜索、详情、个人主页、互动、发布等相关能力。对于这个 skill，应优先使用内容发现和详情读取能力。

目标结果：
- 找出今日热门或快速升温的笔记
- 提取标题、主题、开头钩子、内容形式、情绪调性以及重复出现的 CTA
- 必须基于多篇内容，而不是单篇样本

如果工具不支持严格按“今日”过滤，则使用最新可获得的热门结果，并明确说明“今日风向”是从当前可获取的最新热门内容中归纳出来的。

### 5. 先输出热门风向总结

在询问产品信息之前，先输出一段简短的“热门风向”总结，内容至少包括：
- 3 到 5 个主流主题
- 常见开头钩子结构
- 常见证明方式，例如前后对比、清单、价格、对比测评、日常流程、转变案例
- 常见内容包装形式，例如清单帖、日记体、测评帖、教程帖、避坑帖
- 1 条简短建议，说明哪类产品切入角度最容易贴合当前风向

此时不要直接写最终营销推文。

### 6. 按顺序追问两个问题

完成风向总结后：
1. 先问用户要营销什么产品或服务。
2. 用户回答后，再问想用什么风格来写。

风格示例：
- 种草
- 专业测评
- 真实自述
- 高端克制
- 情绪共鸣
- 爆款标题党

这两个问题必须主动追问，表达要短、直接。

### 7. 生成最终营销推文

只有在“产品”和“风格”都明确后，才可以输出最终中文营销推文。

输出结构：
- `标题`
- `正文`
- `推荐配图方向`
- `推荐话题`

写作要求：
- 严格匹配用户指定的风格
- 文案要建立在前面总结的风向之上，不能写成泛泛的广告词
- 除非用户明确要求设定人设，否则不要编造个人经历或虚假使用感受
- 优先采用“利益点 + 场景 + 证明 + CTA”的清晰顺序
- 语气要贴近小红书平台原生表达

## 失败处理规则

- 如果本地服务无法启动，必须说明启动障碍并停止。
- 如果当前需要登录，不要继续伪造趋势分析。
- 如果当前会话没有可用的 MCP 访问能力，要明确说明“本地服务已启动，但客户端侧未接入 MCP”，然后停止，而不是编造数据。
